# 训练目标 v1 范围与数据契约

训练目标是成员自主管理的私有训练记录。第一版只解决“设定一个可量化目标，并使用可信同步数据持续核对进度”，不引入管理员指导、成员互看、排行或提醒。

## 支持的目标

| 目标类型           | 支持平台                             | 用户填写     | 创建时冻结的基线                           |
| ------------------ | ------------------------------------ | ------------ | ------------------------------------------ |
| 总通过题数增加     | Codeforces、牛客、AtCoder、洛谷、QOJ | 计划增加题数 | 当时具有成功同步数据的平台集合及各平台题数 |
| 单平台通过题数增加 | Codeforces、牛客、AtCoder、洛谷、QOJ | 计划增加题数 | 该平台最近一次成功同步题数                 |
| 单平台 Rating 达标 | Codeforces、牛客、AtCoder、XCPC ELO  | 目标 Rating  | 该平台最近一次成功同步当前 Rating          |

目标从创建当天开始，截止日期必须位于开始日期后的 7–365 天。总题数目标只跟踪创建时进入基线的平台，新绑定的平台不会补入旧目标，也不会凭空抬高进度。目标名称、目标值和截止日期可以编辑；指标、平台、开始日期、基线值和基线组成不可修改。

## 进度与状态

- 进度只读取 `sync_runs.status = 'succeeded'` 对应的最新 `stat_snapshots`。失败、运行中或跳过的同步不能推进或倒退目标。
- 题数低于创建基线时，进度按 0 计算并显示回退提示，不产生负进度。
- 目标跟踪的平台缺少成功快照、解除绑定或不可用时，进度显示“暂不可计算”，缺失值不得按 0 处理。
- 达到 100% 后由成员确认完成；未确认前仍为进行中。超过截止日期的未完成目标显示为已过期。
- 完成和归档都保留历史记录。归档不会删除目标；永久注销账号时目标随 profile 级联删除。

## 隐私与权限

- 普通成员只能读取和操作自己的目标。
- 管理员使用相同入口时也只能管理自己的目标，不获得跨成员读取或修改权限。
- 所有写操作通过从 `auth.uid()` 推导所有者的 RPC 完成，浏览器不能直接写表、基线或生命周期字段。
- 第一版不提供教练授权。将来如需队内指导，必须新增独立授权、撤销和审计设计。
- 个人数据导出包含本人目标、不可变基线和生命周期；导出 RPC 不接受目标成员 ID。

## 第一版不包含

- 公开目标、成员互看、目标排行、点赞或评论。
- 管理员代建、代改、批量分配目标。
- 自动推荐目标、训练计划生成、站内提醒或群机器人通知。
- Rating 曲线、过题明细、标签统计或连续活跃天数。
