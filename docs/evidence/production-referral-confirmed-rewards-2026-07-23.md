# 推荐奖励邮箱确认安全修复 — 2026-07-23

本文只记录脱敏结果，不记录成员身份、邀请码、JWT、数据库密码或原始响应正文。

## 问题与修复

- 独立审查确认旧注册触发器会在 `auth.users` 插入后立即计奖，未验证邮箱也能占用邀请次数并增加 WebChat Token 上限。
- `202607230000_referral_confirmed_rewards.sql` 保留注册时的邀请码校验，但把绑定和计奖延后到邮箱首次确认。
- 邮箱确认触发器使用既有 `profile -> config -> code -> access` 锁序；重复确认不会重复计奖。
- 确认时若计划已暂停、邀请码已失效、邀请人不可用或额度达到上限，奖励会被跳过，但邮箱确认本身不会失败。
- 迁移会撤销仍未确认账号已经产生的绑定、奖励次数和对应 Token 上限，并写入脱敏聚合审计；以后正常确认时仍可按当时状态重新判断资格。
