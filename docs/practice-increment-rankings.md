# 刷题增量榜统计口径

`/rankings` 的刷题榜支持四种范围：累计总数、本周、本月和自定义。增量范围统一使用北京时间自然日；本周从周一开始，本月从当月 1 日开始，自定义范围首尾日期均包含，单次最多 366 天。

## 增量公式

对成员 `i` 和刷题平台 `k`：

1. 在区间开始时刻之前，寻找当前平台绑定生效后最后一次成功同步的累计通过题数 `B_i,k`。
2. 在区间内，寻找最后一次成功同步的累计通过题数 `E_i,k`。
3. 两个观测都存在且 `E_i,k >= B_i,k` 时，平台增量为 `E_i,k - B_i,k`。
4. 累计题数回退时按 `0` 计，并在界面标记“累计题数发生回退，按 0 计”，避免产生负增量。
5. 总榜把各个已成功计算的平台增量相加，同时显示“已统计平台数 / 当前已绑定平台数”。

不会把区间内第一条快照当作基线。首次绑定同步返回的是账号完整历史累计题数，如果直接把它当作区间起点，会把绑定前做过的题误算为本期新增。

## 数据覆盖状态

| 状态             | 含义                                                        |
| ---------------- | ----------------------------------------------------------- |
| 统计完整         | 区间前有成功基线，区间内有成功末值                          |
| 未绑定           | 当前没有该平台的已验证绑定                                  |
| 缺少区间前基线   | 区间开始前没有当前账号绑定下的成功累计值                    |
| 区间内无成功同步 | 有区间前基线，但区间内没有成功观测                          |
| 累计题数发生回退 | 上游修正、管理员纠正或题库变化导致末值小于基线，本期按 0 计 |

失败同步不会作为基线或区间末值。账号换绑后，旧账号的历史快照不会参与新账号增量计算。管理员手工录入会创建成功快照，因此会和自动同步使用同一统计规则。

## 数据库边界

公开客户端只调用 `get_public_practice_increments(date, date)`，不直接读取原始 `stat_snapshots` 或 `sync_runs`。RPC 只返回公开成员 ID、平台、增量、两端累计值、两端观测时间和覆盖状态；QQ、邮箱、角色、停用原因、原始错误和平台私有凭据均不返回。

RPC 对日期顺序、未来日期和最大跨度做服务端复核。查询使用 `stat_snapshots_solved_range_idx` 进行成员、平台和时间范围定位，并通过 `sync_runs.status = 'succeeded'` 排除失败运行。
