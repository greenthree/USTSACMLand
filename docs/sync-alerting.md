# 运行状态与同步失败巡检

生产环境不接入外部告警 Webhook，也不要求配置告警 URL 或 Token。同步、Firecrawl 与 WebChat 的业务行为不依赖外部通知服务。

## 同步失败

- Codeforces、牛客、AtCoder、XCPC ELO、洛谷和 QOJ 的可恢复错误最多自动重试一次。
- 第一次可恢复失败进入两分钟退避，不写成最终失败。
- 第二次仍失败，或首次遇到认证、结构、权限、账号校验等不可重试错误时，任务进入最终失败状态。
- 最终失败保留上一份成功统计，不能把 Rating 或题数写成 `0`。
- 管理员通过后台同步中心和数据源健康页查看平台、错误分类、尝试次数、最后成功时间和数据新鲜度。

## Firecrawl

- 后台显示各 Key 的启用状态、优先级、额度观察、冷却和最近错误。
- 每周 QOJ 或牛客计划批次前可以检查已启用 Key 的额度；单 Key 检查失败不得阻塞其他 Key 或平台。
- 额度或抓取异常通过后台状态巡检，不发送外部通知。

## WebChat

- `/admin/webchat` 显示当天全站请求、已结算 Token、正在预留 Token、剩余额度和北京时间重置时间。
- 请求或 Token 超过预算时仍原子拒绝并返回 `503`，不得自动重试或放行。
- 预算阻断状态保留在私有数据库账本中，由管理员后台查看，不向外部平台投递。

## 运行时错误

Edge Function 使用结构化日志记录固定函数名、固定错误类别、时间和安全格式的请求 ID。日志和后台状态不得包含成员姓名、邮箱、QQ、平台账号、Cookie、Token、消息正文、第三方响应正文或异常堆栈。

浏览器端只提供顶层错误边界和脱敏本地事件，不保存服务端凭据。

## 发布后巡检

1. 查看后台同步中心，确认最近批次完成且队列没有异常堆积。
2. 抽查六个平台的最后成功时间、最终状态和数据新鲜度。
3. 对受控测试任务验证第一次可恢复失败进入退避，第二次失败终止，且没有第三次 attempt。
4. 确认 QOJ 每个 attempt 都关闭临时 Firecrawl 会话，凭据或结构错误不重试。
5. 查看 Firecrawl Key 状态与额度，确认冷却和轮换没有让全部 Key 同时不可用。
6. 查看 `/admin/webchat` 的预算与账本，确认阻断请求返回 `503` 且没有自动重试。
7. 检查日志只使用脱敏字段，不出现 Secret 或成员私有数据。
