# USTS ACM Land 隐私说明

更新日期：2026 年 7 月 18 日

## 收集的数据

注册时使用邮箱、密码和姓名。认证由 Supabase Auth 处理，本项目业务表不读取或保存密码明文。成员资料包括姓名、QQ、年级、专业和竞赛平台账号；同步服务还会保存 Rating、通过题数、更新时间、历史快照和结构化错误状态。浏览器会在当前设备保存未提交的资料草稿，保存成功、退出或注销后清除。

## 公开范围

注册后账号直接启用。姓名、年级和专业填写完整且成员保持公开状态时，已验证的平台账号、Rating、通过题数、数据更新时间，以及由成功历史快照计算的时间范围刷题增量会显示在公开页面。邮箱、QQ、密码、登录令牌、同步错误原文和后台审计详情不进入公开榜单。管理员停用成员后，其资料会从公开页面隐藏并停止同步，但现有账号和业务数据继续保留，直到恢复或永久注销。

## 数据用途

- 确认集训队成员身份并管理平台绑定。
- 生成队内 Rating 榜、刷题榜和成员详情。
- 提供每日题目、成员完成记录和队内讨论。
- 执行定时同步、故障诊断、安全审计和滥用防护。

## 每日题目与讨论

每日题目由管理员维护，题面仍保留在对应竞赛平台，本网站只保存题目链接、标题、难度、标签和训练提示。已登录且账号启用的成员可以记录完成状态并参与讨论；讨论作者只在成员范围内显示，管理员可以隐藏不合适的内容。访客可以查看已发布且已到日期的题目，但不能读取成员完成明细或讨论内容，公开页面只展示匿名聚合数量。

## 第三方服务

GitHub Pages 托管静态前端，Supabase 提供认证、数据库和服务端函数。同步时会使用成员填写的平台账号查询 Codeforces、牛客、AtCoder、AtCoder Problems、XCPC ELO、洛谷和 QOJ；牛客反自动化回退查询和 QOJ 临时登录使用 Firecrawl。QOJ 登录会把专用服务账号和目标用户名发送给 Firecrawl，但不会发送成员邮箱、QQ 或本站密码。完整字段、认证方式和保留边界见[第三方数据来源说明](./docs/third-party-data-sources.md)。各服务对请求数据的处理同时受其自身政策和生产配置约束。

## 修改与删除

成员可以在“我的资料”中更正资料和非 XCPC 平台账号；平台账号变更后需要重新验证。普通成员可在账号页再次验证当前密码并二次确认后永久注销本人账号，无需由管理员代删。管理员账号必须先完成权限交接和降级，再按普通成员流程注销；需要停用账号或处理错误绑定时，可通过集训队内部渠道联系管理员。

永久注销会删除 Auth 用户、资料、平台绑定、当前统计、历史快照和同步记录，并清除相关审计记录中的个人标识，只保留不可关联成员的匿名删除事件。正在运行同步时，注销会整体失败，不会先删除部分数据。删除前，服务端还会在数据库之外保存一个不含成员身份的恢复下限时间；该记录失败时不会执行删除。Supabase/GitHub/Firecrawl 的既有备份与日志可能按生产配置或服务政策继续保留，本站不能改写已经生成的供应商备份，但禁止把早于最新注销恢复下限的备份恢复上线。详细规则见[账号与数据生命周期](./docs/data-lifecycle.md)、[数据库备份与恢复方案](./docs/backup-and-recovery.md)和[生产运维手册](./docs/operations-runbook.md)。

## 安全边界

本项目仅用于集训队训练与展示，不是学校统一身份系统。请勿复用其他网站的密码，也不要在平台账号字段中填写 Cookie、Token 或其他秘密。若发现数据泄露或越权问题，请不要在公开 Issue 中附带敏感信息，应通过 GitHub 私密漏洞报告或集训队内部渠道联系维护者。
