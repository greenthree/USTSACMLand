# USTSACMLand 开发路线图

本文档按“先验证数据源，再搭建业务，再上线”的顺序推进。里程碑编号不等同于发布日期；每个阶段只有在验收条件满足后才能进入下一阶段。

## 范围基线

首个可用版本必须包含：

- Codeforces、牛客、AtCoder、XCPC ELO Rating。
- Codeforces、牛客、AtCoder、洛谷、QOJ 刷题数。
- 邮箱注册后自动登录、密码登录/找回。
- QQ、年级、专业、姓名和各平台账号维护。
- 平台账号验证、成员数据管理、手动同步、同步日志与数据源状态。
- GitHub Pages 公开站点和安全的服务端数据同步。

不属于首版阻塞项：比赛提醒、群机器人播报、题目明细浏览、战队训练计划、移动端原生应用。

建议同步频率：

| 数据源                             | 默认频率                  | 数据过期阈值 | 说明                                       |

M3 验收条件：

- 定时同步连续运行 7 天，无秘密泄漏、无限重试或重复快照。
- 单个平台故障不会阻止其他平台更新。

## 主要风险清单

| 风险                          | 影响                                 | 应对                                           |
| ----------------------------- | ------------------------------------ | ---------------------------------------------- |

## 完成定义

任一功能只有同时满足以下条件才可标记完成：

- 行为符合验收标准，正常、空数据、失败和无权限状态均有处理。
- 关键逻辑有自动化测试，数据源解析有固定样本契约测试。
- 不引入明文秘密、越权查询或未审计的管理员写操作。
- 文档、数据库 migration 和生成类型与实现保持一致。
- 在 GitHub Pages 生产构建中验证，而不只在本地开发服务器运行。
